# reference 目录说明

`reference/` 只用于“定制和查细节”，不是主入口。

## 你通常不需要看 reference

日常运行请直接看 `../SKILL.md` 并执行主脚本。

## 什么时候看 reference

- 需要改建表逻辑：看 `code/30_create_pricing_schema.md` 和 `../scripts/30_create_pricing_schema.dos`
- 需要改拉数与入库逻辑：看 `code/50_build_and_ingest_api_2026.md` 和 `../scripts/50_build_and_ingest_api_2026.py`
- 需要核对 API 字段定义：看 `API/*.pdf`
