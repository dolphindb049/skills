# 文档自动化翻译技能 (Document Translation Skill)

这个技能提供一套强大的 PDF 与 Word 文档自动化批量翻译方案。本工具的核心逻辑是**逐段/逐句提取**文档内的文字，单独发送给大模型进行纯文本翻译，拿到结果后再**原位覆盖写回**。这种方式极大程度上避免了全篇扔给模型导致输出乱码丢字的问题，**完美保留了原文档的排版结构、字体格式与复杂表格**。

该工具具备以下优异特性：
1. **双向翻译覆盖**：不仅支持**中文内容自动化翻译为英文**，同时也提供一键反转，支持将**英文文献/内容翻译为纯正中文**。
2. **多线程进度可见**：控制台提供针对段落数级别的进度条（tqdm），精准预估并支持安全中断（Ctrl+C）。
3. **格式修复**：自带字体统一样式能力，解决 PDF 转 Word 时可能出现的中英文字体大小不一的排版异常。

## ✨ 核心组成

你将在该目录下发现两个核心 Python 脚本：
- `pdf_docx.py`：负责将 PDF 文档零损耗（结合字体修复）转换为可编辑的 DOCX 文档。
- `translate_docx.py`：负责对 DOCX 文档进行递归式的逐句定位和文本翻译重写。

## 🛠️ 前置环境与文件夹设置

推荐使用新时代的包管理工具 `uv` 配置极速且干净的环境。你需要执行以下命令：

```shell
# 1. 创建并激活虚拟环境 (需预先在系统中装有 uv)
uv venv
# Windows 激活虚拟环境
.venv\Scripts\activate

# 2. 安装此工具相关的核心依赖
uv pip install pdf2docx python-docx openai tqdm
```

**工作目录结构（系统会自动建立以下文件夹进行调度）：**
* 📁 `originfiles/`：**源 PDF 放置处**。如果你上来拿到的就是 PDF 扫描件或文献，先丢进这里。
* 📁 `docxfiles/`：**待翻译的 Word 放置处**。如果是刚转换好的文件会吐在这个目录；如果你手里直接有 Word 原档，也可以不用管上面，**直接放在这里**。
* 📁 `finishfiles/`：**最终翻译产物**。翻译后的成品统一输出在这里。

## 🚀 核心执行流程 (一二三步)

你只需要判断当前手中文件的格式，就可以按照对应的顺序干活：

### 场景 A：如果你的源文件是 PDF（需要变 DOCX 再翻译）

- **第一步：放置文件**。把需要处理的一批 PDF 文件放到 `originfiles/` 文件夹中。
- **第二步：转换格式**。在控制台运行预处理脚本变作 DOCX：
  ```shell
  uv run pdf_docx.py
  ```
  *(完成后，你会发现 `docxfiles/` 中出现了新生的同名 Word 文档)*
- **第三步：开始翻译**。打开 `translate_docx.py` 确认你想要的 `TARGET_LANGUAGE` （设为 "en" 是翻译为英文，"zh" 代表翻译为中文），然后执行：
  ```shell
  uv run translate_docx.py
  ```
  坐等进度条跑完后，去 `finishfiles/` 取最终文档即可！

### 场景 B：如果你的源文件已经是 DOCX（直接翻译）

- **第一步：放置文件**。不需要做转换，**直接丢过第一和第二步**，将现成的 `.docx` 文件拷进中间文件夹 `docxfiles/` 里面。
- **第二步：设置语言**。用代码编辑器打开并检查 `translate_docx.py`，确认文件顶部的配置项：
  - 中文翻译成英文： `TARGET_LANGUAGE = "en"` 
  - 英文翻译成中文： `TARGET_LANGUAGE = "zh"` 
- **第三步：开始翻译**。在终端运行批量翻译工具：
  ```shell
  uv run translate_docx.py
  ```
  进度条加载完毕之后，进入 `finishfiles/` 提取完美保留样式的熟肉文档！
