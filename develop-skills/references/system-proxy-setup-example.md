# Reference: 系统代理配置 Skill 演进示例 (Meta-Reference)

这是一个辅助演示如何把从复杂 Task 中“踩坑”抽象成通用 Skill 的演进例子。

### 阶段一：纯粹的 Task 执行记录（这不是一个好 Skill，因为它不普适）

> ⚠️ Bad Code
> 今天我要在一台新买的阿里云服务器（IP 192.168.1.100）上安装 Clash。
> 我先 `wget http://xxxx`，然后发现网络不通。
> 我去找了 IT，拿到了内网跳板机。
> 把某大神的 config 拷贝过去。
> 发现 `Permission Denied`，然后加了 `chmod +x`。
> 然后跑不起来，发现 7890 端口被 docker 占用了。最后我改成了 8890。好了。

### 阶段二：提取复用的方法论形成 Skill (Doc as Skill)

> ✅ Good Method
> **Skill Name**: Linux环境自动化部署与调试通用代理服务(Clash)
> 
> **工作流抽象**：
> 1. 环境准备检查：由于企业内网可能限制外网 wget，**优先尝试**使用镜像源或挂载内部共享盘符，不要盲目阻塞于外传。
> 2. 权限处理通用化：所有 Linux 二进制构建落地后，均需包含规范的 `chmod +x` 及属主查验。
> 3. 冲突规避：直接运行容易遇到端口抢占，应当执行预检 `netstat -tulnp | grep 7890`，若被占用，使用动态端口分配逻辑而非写死。
> 
> 后续所有的这类任务通过此规则，不仅解决了这台机器配代理的问题，还可以覆盖配 NGINX、配各类其它网络插件的需求。这就是从 Task 迈向了具有可演进性的 Skill。
